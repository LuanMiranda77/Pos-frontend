Introdução

Após entrevista identificamos que os setores de pecuária precisava ser automatizado devido à não ter um controle do que é produzido, nem do que é vendido, com isso surgiu a necessidade de um sistema capaz de facilitar o gerenciamento da cooperativa IFET. O sistema será capaz de auxiliar no gerenciamento da produção até a venda, com relatórios financeiros, controle de estoque, central de venda, tudo em um único ambiente separa por níveis de usuários, garantindo a segurança dos dados e das informações financeiras.

Objetivo

Desenvolver um sistema de gerenciamento para a cooperativa do Instituto Federal do Sudeste de Minas que automatize e otimize os processos de produção e venda de produtos de uma variedade de setores, como horto, suínos, laticínios e aves. O sistema permitirá que os gestores, funcionários e vendedores registrem, verifiquem e gerenciem todas as etapas da produção e venda de maneira fácil de entender e centralizada.

Tecnologia

Uma arquitetura que tem demonstrado ser flexível e poderosa neste contexto combina um banco de dados PostgreSQL robusto, uma API criada com Spring Boot e interfaces de usuário (UI) criadas com React para aplicativos web e React Native para aplicativos mobile.

Arquitetura de software

Banco de dados

O banco de dados PostgreSQL é o núcleo dessa arquitetura. O PostgreSQL é um sistema de gerenciamento de banco de dados relacional de código aberto com recursos abundantes, escalabilidade e confiabilidade. Devido à sua capacidade de gerenciar grandes quantidades de dados de forma segura e eficiente, ele é uma escolha popular para armazenamento e recuperação de dados em aplicativos. Sua adaptabilidade o torna adequado para uma ampla gama de aplicações, incluindo aplicativos comerciais e sistemas de análise de dados.

Back-end

Em cima dessa base de dados confiável, criamos os serviços com o Spring Boot. O Spring Boot é um framework Java que facilita a criação de micro-serviços e APIs RESTfull. Ele fornece recursos poderosos para a criação de serviços web seguros e escaláveis. O Spring Boot permite a gestão de solicitações HTTP, administração do acesso a dados no PostgreSQL e criação de endpoints que comunicam com aplicativos web e móveis.

Front-end

Existem duas frentes para a criação da interface do usuário: React para aplicativos web e React Native para aplicativos mobile. React é uma biblioteca JavaScript de código aberto mantida pelo Facebook que permite a criação de interfaces de usuário interativas. O React permite a criação de aplicativos web modernos com uma experiência de usuário rápida e atraente quando usado no desenvolvimento web. Por outro lado, o React Native expande essa capacidade para aplicativos mobile, permitindo que os desenvolvedores reduzam o tempo e o esforço de desenvolvimento reutilizando uma grande parte do código entre as plataformas Android e iOS.

Essa arquitetura tem muitos benefícios. Primeiro, a divisão do front-end e do back-end permite que as equipes trabalhem separadamente e se concentrem em suas áreas de especialização. Segundo, a escolha de tecnologias confiáveis e populares, como PostgreSQL, Spring Boot, React e React Native, garante escalabilidade, estabilidade e segurança do sistema. Terceiro, a possibilidade de compartilhar código entre aplicativos móveis e web acelera o processo de desenvolvimento e mantém a experiência do usuário concisa.
